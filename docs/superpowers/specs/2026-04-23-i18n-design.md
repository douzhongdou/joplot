# 多语言接入设计

日期：2026-04-23

## 目标

为项目接入基于字典的多语言方案，首批支持：

- 中文 `zh-CN`
- 英文 `en`
- 日文 `ja-JP`

要求：

- 跟随浏览器语言初始化
- 支持用户手动切换
- 语言选择持久化到 `localStorage`
- 只翻译系统文案，不自动翻译用户可编辑内容

## 方案

采用轻量自建 i18n：

- `src/i18n/config.ts`
  - 支持语言列表
  - 浏览器语言归一化
  - 初始语言解析
- `src/i18n/dictionaries/*`
  - 三份字典文件
- `src/i18n/index.tsx`
  - `I18nProvider`
  - `useI18n()`
  - `t(key, params)`
  - `formatNumber(value)`

## 接入范围

首批接入系统文案：

- navbar 语言切换
- 首页空状态
- 上传按钮与拖拽浮层
- WorkbenchHeader
- CardInspector
- ChartCard
- SelectMenu 默认 placeholder
- 上传提示工具 `src/lib/upload.ts`

## 边界

以下内容不自动跟随语言切换：

- 用户手动编辑过的图卡标题
- 用户手动编辑过的系列显示名
- 数据文件名
- CSV 字段名

说明：

- 这些内容属于用户数据或导入数据，不应被系统翻译覆盖。
- 系统自动新建的默认标题和复制后缀会按当前语言生成，但生成后即视为卡片内容的一部分。

## 语言切换规则

初始化优先级：

1. `localStorage` 中的语言
2. 浏览器语言
3. 默认回退到 `en`

语言切换入口：

- 放在 navbar 右侧

## 测试

新增测试覆盖：

- 语言归一化
- 初始语言解析优先级
- 上传提示多语言返回

保留原有：

- `workbench`
- `upload`
- `plotViewport`
- `clipboard`
