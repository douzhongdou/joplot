# Joplot 移动端优化设计

## 目标

让 Joplot 在手机尺寸下保持“画布优先、底部操作”的工作方式。用户应当能够在移动端完成：

- 通过显式按钮上传文件
- 在图表视图和数据视图之间切换
- 打开底部操作面板完成上传、添加组件、筛选和重置
- 选中图卡后通过底部抽屉修改配置
- 下载或复制图表

同时保持桌面端现有工作流不被破坏。

## 当前问题

1. 顶部仍沿用桌面端的信息架构，`navbar` 和工作台工具条叠加后，手机首屏被压缩得过于严重。
2. 移动端画布下方使用整宽“编辑当前图卡”按钮，视觉重量过大，但信息增量有限。
3. 上传、添加组件、筛选、重置这些动作虽然已经显式出现，但放在页面顶部会长期占据有限的纵向空间。
4. 工作台目前仍是“把桌面端控件搬到手机上”的思路，缺少更符合拇指区的底部导航和底部面板。

## 方案

### 1. 顶部极简 Header

- 桌面端保持现有头部结构不变。
- 移动端顶部只保留：
  - `logo`
  - 语言切换
  - 帮助入口
- `图表 / 数据` 视图切换不再占用顶部导航位置。
- 移动端顶部高度控制在接近原生 App 工具栏的密度，避免 logo 区继续挤压画布。

### 2. 底部导航栏

- 仅在移动端且已有数据时显示底部导航栏。
- 底部导航提供三个一级入口：
  - `图表`
  - `数据`
  - `操作`
- `图表 / 数据` 负责切换主内容视图。
- `操作` 不直接切页，而是拉起底部操作面板。
- 导航栏应保持紧凑，以 icon + 短标签为主，不使用厚重外框。

### 3. 底部操作面板

- 移动端不再在页面顶部长期展示工作台工具条。
- 点击底部导航里的 `操作` 后，使用底部面板承载现有工作台能力。
- 底部面板内部复用 `WorkbenchHeader` 的核心逻辑，提供：
  - 上传数据
  - 添加组件
  - 筛选
  - 重置数据
- 这样可以保留完整能力，同时把首屏空间还给图表画布。

### 4. 图卡编辑入口

- `CardInspector` 继续通过底部抽屉承载。
- 但移动端触发入口从整宽大按钮改为右下角小号悬浮按钮。
- 这个按钮只在：
  - 当前处于图表视图
  - 已选中图卡
  - 设备为移动端
  时出现。
- 文案缩短为更轻量的“编辑”，避免底部视觉过重。

### 5. 图卡与画布

- 移动端继续采用单列全宽图卡堆叠。
- 不强调复杂拖拽栅格体验，拖拽和缩放句柄在移动端关闭。
- 为底部导航和悬浮编辑按钮预留安全区，避免遮挡图表内容。
- 图卡内部标题、标签和导出操作继续保留，但优先保证阅读空间和触摸命中率。

## 实现边界

- 不重做整体品牌视觉风格。
- 不改图表数据模型和业务逻辑。
- 不新增后端能力。
- 不为移动端单独引入新的全局状态管理。
- 主要改动集中在布局壳层、导航承载和移动端交互组织方式。

## 验证标准

1. 手机宽度下顶部只保留轻量全局头部，不再出现两层导航叠加。
2. 手机宽度下 `图表 / 数据 / 操作` 通过底部导航完成。
3. 顶部工作台工具条不再长期占据首屏。
4. 选中图卡后通过小号悬浮按钮打开编辑抽屉。
5. 上传、添加组件、筛选、重置在移动端仍可完成。
6. 桌面端现有布局和交互基本保持不变。
