# 多语言 SEO 设计

日期：2026-04-23

## 目标

为 `joplot` 提供可被搜索引擎稳定索引的多语言入口，而不是仅靠运行时切换语言。

本次目标：

- 输出独立语言路径：
  - `/zh`
  - `/en`
  - `/ja`
- 根路径 `/` 按浏览器语言跳转到对应语言页
- 仅语言路径页参与索引，根路径作为跳转入口不参与主收录
- 每个语言页具备独立的：
  - `html lang`
  - `title`
  - `description`
  - `canonical`
  - `hreflang`
  - 本地化结构化数据
- 语言切换器同步切换 URL，而不是仅修改 `localStorage`

## 现状问题

当前项目已经支持字典驱动的多语言，但 SEO 仍有几个明显短板：

- 只有一个入口页，搜索引擎无法稳定区分中英日三种版本
- 语言依赖浏览器语言和 `localStorage`，URL 本身不表达语言语义
- 页面元信息只有中文版本，不利于英文和日文索引
- 语言切换后 URL 不变，分享链接与收录链接都不稳定

## 方案

采用“保留现有 React/Vite SPA + 增加静态多入口”的方案。

### 1. 构建输出

通过 Vite 多入口构建输出四类页面：

- 根路径入口：`/`
- 中文页：`/zh`
- 英文页：`/en`
- 日文页：`/ja`

实现方式：

- 根路径使用独立 `index.html`
- 三个语言页分别使用：
  - `zh/index.html`
  - `en/index.html`
  - `ja/index.html`

这样不需要迁移框架，也不需要引入 SSR，即可直接得到可索引的静态语言入口。

### 2. 根路径策略

根路径 `/` 只负责浏览器语言跳转：

- 浏览器语言以 `zh` 开头时跳转 `/zh`
- 浏览器语言以 `ja` 开头时跳转 `/ja`
- 其他情况默认跳转 `/en`

同时：

- `/` 标记为 `noindex`
- 保留无脚本备用语言链接

### 3. 语言来源优先级

应用初始化语言优先级调整为：

1. 当前路径对应语言
2. `localStorage`
3. 浏览器语言
4. 默认回退到 `en`

说明：

- SEO 页面的语言必须首先由 URL 决定
- `localStorage` 仍保留，用于记忆用户主动切换
- 语言切换时要同步跳转到对应语言路径

### 4. 页面元信息

每个语言页提供独立的：

- 页面标题
- 页面描述
- Open Graph 标题与描述
- Twitter 标题与描述
- `canonical`
- `hreflang`

本次采用静态写入，保证首屏 HTML 就带有正确语言元信息。

### 5. 语言切换

导航栏中的语言切换器从“只改语言状态”调整为“切换到对应语言 URL”。

例如：

- 当前位于 `/zh`
- 用户切换到英语
- 页面跳转到 `/en`

这样可以保证：

- 用户复制链接时语言明确
- 搜索引擎看到的链接稳定
- 页面语言与 URL 不会漂移

## 边界

本次不做：

- SSR / SSG 框架迁移
- 动态 sitemap 域名生成
- 多地区拆分（如 `en-US`、`en-GB`）
- 每个功能页的二级语言路由体系

本次只解决首页级多语言 SEO 和语言入口稳定性问题。

## 验证

完成后需要验证：

- `vite build` 成功
- `dist/index.html` 为跳转入口
- `dist/zh/index.html`、`dist/en/index.html`、`dist/ja/index.html` 存在
- 三个语言页包含对应语言元信息
- 语言切换器切换后 URL 随之变化

## 后续建议

- 等正式线上域名稳定后，将 `canonical` 和 `hreflang` 的相对路径升级为绝对 URL
- 增加 `sitemap.xml`
- 将首页正文中的核心卖点文案继续按语言精修，以提升不同语种的搜索匹配度
