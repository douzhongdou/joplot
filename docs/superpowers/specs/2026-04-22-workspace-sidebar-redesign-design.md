# 工作台与右侧配置抽屉重构设计

## 背景

现有界面把数据集、筛选、工作台操作、图卡配置全部堆在右侧侧栏中，导致：

- 视觉层级混乱，工作台级操作与图卡级操作混在一起
- 多个卡片式分区重复描边，占用大量纵向空间
- 多 CSV 叠图时，系列绑定需要逐条手动选择，操作成本过高
- 显示相关配置与数据绑定混杂，缺少清晰的职责分离

本次重构以“类似多维表格仪表盘的工作区分层”为参考，但不做 1:1 复刻。

## 目标

- 主窗口顶部承载工作台级操作：添加组件、数据源、筛选
- 右侧侧栏只保留当前选中图卡的配置抽屉
- 图卡配置拆分为 `基础配置` 与 `显示设置` 两个 tab
- 多 CSV 拖入后，创建多系列图表时自动按导入顺序绑定数据集
- 系列编辑改为更紧凑的两行结构，减少无意义的卡片容器

## 信息架构

### 工作台级区域

位于主窗口顶部，负责：

- `添加组件`
- 当前数据源选择
- 数据范围概览
- 筛选条件摘要与编辑入口

### 图卡级区域

位于右侧配置抽屉，负责当前选中图卡：

- 基础配置
- 显示设置
- 图卡复制与删除

## 交互设计

### 添加组件

- `添加组件` 放在主窗口顶部
- 点击后弹出组件类型菜单
- 支持：折线图、散点图、柱状图、统计卡

### 数据源与筛选

- 数据源选择移到顶部上下文区域
- 筛选条件以摘要 chip 展示
- 点击 `筛选` 按钮展开筛选编辑面板

### 多 CSV 自动绑定

- 当用户拖入多个 CSV 后，新建多系列图表时：
  - 默认使用当前活动数据集的首列作为公共 X 轴
  - 按导入顺序自动为各数据集生成系列
  - 系列默认自动绑定对应数据集和默认数值字段
- 手动添加系列时，优先补齐尚未绑定的数据集

## 右侧抽屉设计

### 基础配置 Tab

包含：

- 标题
- 图表类型
- 公共 X 轴
- 绘制方式
- 线宽
- 数据系列

#### 数据系列

- 一条系列固定为两行结构：
  - 第一行：系列标题 + `...`
  - 第二行：数据源选择 + 字段选择
- 不再使用卡片包裹单条系列
- `...` 菜单用于删除系列或跳转到显示设置

### 显示设置 Tab

包含：

- 系列显示
  - 显示名
  - 颜色
- 图表显示
  - 图例
  - 网格线
  - 坐标轴
  - 主题色（入口保留，当前禁用）

#### 颜色入口

- 使用 `色块 + HEX + 调色板 icon`
- 不使用下拉箭头，避免误导为普通 select

## 非目标

- 本次不实现 `分析` tab
- 本次不实现主题映射或 map 主题系统
- 本次不处理构建产物过大的拆包优化

## 验证标准

- `pnpm build` 通过
- 多 CSV 导入后，新建多系列图表时无需逐条手动选择数据集
- 顶部可完成数据源与筛选操作
- 右侧抽屉可在 `基础配置 / 显示设置` 间切换
- 显示设置中的颜色入口是可理解的颜色配置入口
